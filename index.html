  3. django环境

Django 基础命令

  1. startproject 启动项目
  2. startapp 启用应用
  3. check 校验项目完整性
  4. runserver 进入django环境 并运行本地django项目
  5. shell 进入djangoshell
  6. test 跑一些测试用例
  7. makeigrations 创建模型变更的迁移文件
  8. migrate 执行上一个命令创建的迁移文件

Django 项目目录介绍

  1. 项目配置文件：settings.py
  2. 项目路由配置文件：urls.py
  3. 项目管理文件：manage.py

Django应用目录介绍

  1. views.py 视图处理的地方

  2. models.py 定义应用模型的地方

  3. admin.py 定义admin模块管理对象的地方

  4. apps.py 声明应用的地方

  5. tests.py 编写应用测试用例的地方

  6. urls.pu 管理路由的地方 需要自行创建

Django路由

  1. 请求无法达到刚才的视图函数
  2. 通过配置路由绑定视图函数和URL

调用顺序

  1. 通过在views中编写的模块函数
  2. 然后再应用的urls文件中编写应用路由，指向我们编写的模块函数
  3. 再修改项目路由中的urls文件，设置路由，到应用路由中寻找指定模块函数
  4. 就会显示我们设置的网页了

Django模型

  1. 模型层简介

    1. 模型层是什么

      从数据库取出数据进行页面展示，结合视图和数据进行开发。

      模型层位于Django视图层和数据库之间 ，将python对象和数据库表之间转换

  1. 为什么需要模型层

    1. 模型层可以屏蔽不同数据库之间的差异，使开发者可以专注于业务逻辑开发
    2. 提供更多的便捷工具有助于开发

  2. 模型层相关配置

  1. 创建博客文章模型

    1. 设计博客模型
      1. 文章标题 文本类型
      2. 文章摘要 文本类型
      3. 文章内容 文本类型
      4. 唯一ID标记 int数字类型（自增，主键）
      5. 发布日期 日期类型
    2. 模型层定义字段
      1. 数字类型：integerfield
      2. 文本类型：textfield
      3. 日期类型：datetimefield
      4. 自增id： AutoField
      5. 主键定义：primary_key属性

  2. 初识Django shell

    1. 什么是Django shell
      1. python shell ，用于交互式的python编程
      2. Django shell 也类似，继承Django项目环境
    2. 为什么需要Djangoshell
      1. 临时性操作使用Django shell 更加方便
      2. 小范围Debug更简单，不需要运行整个项目
    3. Django shell 使用
      1. python manage.py shell

  3. 初识Django Admin模块

    1. Django Admin模块是什么
      1. Django模块的后台管理工具，不需要用户开发
      2. 读取定义的模型元数据，提供强大的管理使用页面
    2. 为什么需要使用Django Admin模块
      1. Django shell 新增文章太复杂了
      2. 管理页面时基础设施中重要的部分
      3. 管理页面功能：认证用户、显示管理模块、校验输入等功能
    3. Django Admin模块使用
      1. 创建管理员用户
        1. python manage.py createsuperuser
      2. 登陆页面进行管理
